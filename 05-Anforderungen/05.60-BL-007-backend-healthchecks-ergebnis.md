# EMC Mitgliederverwaltung – P1 / BL-007 Backend Healthchecks

Status: Abgeschlossen
Datum: 2026-06-12

---

# Zielsetzung

Ziel von BL-007 war die Einführung standardisierter Backend-Healthchecks zur Verbesserung von Betrieb, Monitoring, Deployment-Sicherheit und Fehlerdiagnose.

Zusätzlich sollte die technische Transparenz des laufenden Backends verbessert werden, insbesondere hinsichtlich:

- Backend-Version
- Laufzeitumgebung
- aktiver Spring-Profile
- Build-Zeitpunkt

Die Umsetzung sollte mit den Ergebnissen der EMC NAS Betriebsstandardisierung kompatibel sein und die bestehende Uptime-Kuma-Monitoring-Strategie ergänzen.

---

# Ausgangssituation

Vor Umsetzung von BL-007 existierten:

- keine standardisierten Health-Endpunkte
- keine Liveness-Probes
- keine Readiness-Probes
- keine Environment-Erkennung
- keine Build-Time-Informationen
- keine Monitoring-Integration auf Anwendungsebene

Monitoring konnte lediglich Container- und Netzwerkverfügbarkeit bewerten.

Eine Aussage über die tatsächliche Betriebsbereitschaft der Anwendung war nicht möglich.

## Security

Health-Endpunkte wurden bewusst öffentlich freigegeben:

```text
/actuator/health/**
```

Begründung:

- Monitoring benötigt keinen Benutzerlogin
- Veröffentlichung erfolgt ausschließlich technischer Statusinformationen
- keine fachlichen oder personenbezogenen Daten enthalten

Der bestehende SystemInfo-Endpunkt bleibt authentifizierungspflichtig.

# Besondere Erkenntnisse

## Portainer Environment Variables

Während des PROD Deployments wurde festgestellt:

```text
SPRING_PROFILES_ACTIVE
```

war im Container leer, obwohl die Compose-Datei bereits angepasst worden war.

Ursache:

Portainer verwaltet Stack-Environment-Variablen separat.

Erkenntnis:

Neue Compose-Variablen müssen zusätzlich in den Portainer Stack Environment Variables gepflegt werden.

Die Environment-Erkennung von BL-007 hat diesen Konfigurationsfehler unmittelbar sichtbar gemacht.

---

# Architekturgewinn

Durch BL-007 wurde die Anwendung von reinem Container-Monitoring auf echtes Anwendungs-Monitoring erweitert.

Vorher:

```text
Container läuft
```

Nachher:

```text
Anwendung läuft
+
Datenbank erreichbar
+
Monitoring nachweisbar
+
Deployment verifizierbar
```

Dadurch wurde die Betriebsqualität der EMC Mitgliederverwaltung deutlich verbessert.
